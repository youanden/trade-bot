<objective>
Fix the HMAC authentication bug in the Polymarket CLOB client, extract signing into a testable module, and add structured HTTP error handling.

Purpose: The current HMAC implementation has two bugs (secret not base64-decoded, output not URL-safe) that cause authentication failures on all authenticated CLOB endpoints. Structured errors enable callers to distinguish retryable from permanent failures.

Output: `hmac.ts` (standalone signing), `errors.ts` (ClobApiError class), updated `client.ts` using both, and unit tests proving correctness.
</objective>
