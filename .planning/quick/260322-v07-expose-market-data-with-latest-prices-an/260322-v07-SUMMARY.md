## What Was Built

**GET /markets** now fetches all rows from the `prices` table after querying markets, groups them in JS by `marketId`, and finds the latest entry per market using ISO-8601 lexicographic comparison. Each market response object is enriched with `yesPrice`, `noPrice`, and `volume` fields (null when no price row exists).

**POST /markets/sync** is registered before `GET /:id` so Hono does not match the string "sync" as an `:id` path param. It attempts to build an `ExchangeClient` for each platform (`polymarket`, `kalshi`) in a try/catch, skipping any that throw due to missing credentials. If no clients are available, it returns `{ synced: 0, error: "..." }`. Otherwise it creates a `MarketResolver` and calls `syncMarkets(limit)`.

**UI changes:**
- `api.ts`: `syncMarkets()` method added
- `useMarkets.ts`: `refetchInterval: 30_000` enables auto-refresh
- `Markets.tsx`: Sync button with disabled/loading state, `onSuccess` cache invalidation, price fields renamed from `yes_price`/`no_price` to `yesPrice`/`noPrice`, volume row added
