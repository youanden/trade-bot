## Directory Purposes

**`src/worker/api/routes/`:**
- Purpose: One Hono sub-app per REST resource, mounted at `/api/<resource>` in `index.ts`
- Contains: Route handlers that query D1 or call DO stubs; no business logic
- Key files: `bots.ts` (CRUD + DO lifecycle), `analytics.ts` (metrics endpoints)

**`src/worker/bots/`:**
- Purpose: Bot Durable Object lifecycle and all strategy implementations
- Contains: Abstract base, concrete DO, strategy registry, one directory per strategy
- Key files: `base.ts` (alarm loop, trade recording), `registry.ts` (strategy map), `bot-do.ts` (tick dispatch)

**`src/worker/bots/<strategy-name>/`:**
- Purpose: Self-contained strategy module
- Contains: `config.ts` (TypeScript config interface extending `BotConfig`), `strategy.ts` (exported tick function)
- Pattern: Every strategy exports a single `<name>Tick` function of type `StrategyTickFn`

**`src/worker/core/`:**
- Purpose: Shared infrastructure — all code here is platform-agnostic and reusable across strategies
- Sub-modules: `db/`, `exchanges/`, `market/`, `risk/`, `utils/`

**`src/worker/core/exchanges/`:**
- Purpose: Abstracts Polymarket and Kalshi into a single `ExchangeClient` interface
- Contains: Interface definition, platform clients, factory, WebSocket event adapters
- Key files: `types.ts` (interface), `factory.ts` (construction from env)

**`src/worker/core/risk/`:**
- Purpose: All position sizing and risk management logic
- Contains: `PortfolioRisk` class, Kelly functions, analytics computations, per-bot limit configs

**`src/ui/lib/api.ts`:**
- Purpose: Single typed client for all backend calls; used by hooks and page components
- Pattern: Named `api` object with one function per endpoint

**`drizzle/`:**
- Purpose: Generated SQL migration files; committed to version control
- Generated: Yes (by `drizzle-kit generate`)
- Committed: Yes

**`dist/`:**
- Purpose: Vite build output for UI; served as Worker static assets
- Generated: Yes
- Committed: No (build artifact)

## Key File Locations

**Entry Points:**
- `src/worker/index.ts`: Worker fetch handler — mounts all API routes, exports `BotDO`
- `src/ui/main.tsx`: React SPA mount point
- `src/worker/bots/bot-do.ts`: Durable Object class registered in `wrangler.toml`

**Configuration:**
- `env.d.ts`: Global `Env` interface — all Worker bindings and secret names
- `wrangler.toml`: Cloudflare deployment — D1 binding, DO binding, AI binding, assets directory
- `vite.config.ts`: UI build config — `@ui` path alias, proxy `/api` to `localhost:8787`
- `drizzle.config.ts`: Drizzle-kit migration config

**Core Logic:**
- `src/worker/core/db/schema.ts`: Authoritative database schema — all tables and column types
- `src/worker/bots/registry.ts`: Strategy registry — add new strategies here
- `src/worker/bots/base.ts`: `BaseBotDO` — alarm loop, `recordTrade()`, position upsert, audit log
- `src/worker/core/exchanges/types.ts`: `ExchangeClient` interface — contract all exchange clients must satisfy
- `src/worker/core/risk/portfolio.ts`: `PortfolioRisk` — primary risk gate for all strategies

**Testing:**
- `test/core/kelly.test.ts`: Unit tests for Kelly sizing functions
- `test/core/analytics.test.ts`: Unit tests for Sharpe, drawdown, profit factor calculations
- `test/core/matcher.test.ts`: Unit tests for `MarketMatcher` title similarity

## Naming Conventions

**Files:**
- Worker source files: `camelCase.ts` (e.g. `bot-do.ts`, `cross-arb` uses kebab-case directory names)
- Strategy directories: `kebab-case` matching the registry key string (e.g. `cross-arb`, `llm-assessor`)
- Config files: `config.ts` (always this name within a strategy directory)
- Strategy files: `strategy.ts` (always this name within a strategy directory)
- UI pages: `PascalCase.tsx` (e.g. `BotDetail.tsx`, `Analytics.tsx`)
- UI hooks: `camelCase.ts` with `use` prefix (e.g. `useBots.ts`)

**Directories:**
- Strategy modules: kebab-case matching the `strategies.set()` key in `registry.ts`
- Core sub-modules: lowercase single-word (e.g. `db`, `risk`, `market`, `utils`)
- Exchange sub-modules: lowercase platform name (e.g. `polymarket`, `kalshi`)

## Where to Add New Code

**New Strategy:**
1. Create directory: `src/worker/bots/<strategy-name>/`
2. Add `config.ts` — define `interface <Name>Config extends BotConfig { ... }`
3. Add `strategy.ts` — export `async function <name>Tick(bot: BaseBotDO, env: Env): Promise<void>`
4. Register in `src/worker/bots/registry.ts`: `strategies.set('<strategy-name>', <name>Tick)`
5. Import strategy function at top of `registry.ts`

**New API Route:**
1. Create `src/worker/api/routes/<resource>.ts` with a `new Hono<{ Bindings: Env }>()` app
2. Add `app.route('/api/<resource>', <resource>Routes)` in `src/worker/index.ts`

**New Database Table:**
1. Add table definition to `src/worker/core/db/schema.ts`
2. Run `drizzle-kit generate` to create migration file in `drizzle/`
3. Apply migration with `wrangler d1 migrations apply`

**New Exchange Client:**
1. Create `src/worker/core/exchanges/<platform>/client.ts` implementing `ExchangeClient`
2. Add `types.ts` and `websocket.ts` as needed
3. Add construction logic to `src/worker/core/exchanges/factory.ts`
4. Add required credentials to `env.d.ts` and `wrangler.toml [vars]`

**Utilities:**
- Shared helpers accessible to all strategies: `src/worker/core/utils/`
- Risk/sizing functions: `src/worker/core/risk/`
- Market data utilities: `src/worker/core/market/`

**New UI Page:**
1. Add `src/ui/pages/<PageName>.tsx`
2. Add a `<Route>` in `src/ui/App.tsx`
3. Add a `<NavItem>` to the nav in `src/ui/App.tsx` if it needs navigation
4. Add any data-fetching hooks to `src/ui/hooks/`
5. Add API call functions to `src/ui/lib/api.ts`
